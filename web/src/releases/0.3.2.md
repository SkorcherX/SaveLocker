# v0.3.2

_Released 20 July 2026_

Four save-path bugs found on real hardware, plus templated save paths. Two of them could destroy
saves silently while reporting success — if you are on 0.3.0 or earlier, upgrade.

## New

- **Templated save paths.** A game's suggested save folder can now be written as a template such as
  `<winPublic>/Documents/My Game`, and every machine expands it against its own folders. Two machines
  can no longer disagree about where a game's saves live.
- **Agents describe their own save location.** An agent that already has a working folder produces a
  template automatically by reversing its own path resolver — under Proton, against the game's own
  prefix. The first machine you set up correctly teaches the rest of the fleet.
- **One-click templating in the console.** A machine's concrete save path can be turned into a
  template from the game's detail panel.

## Fixed

- **A restore could nest a save folder under itself and delete the correctly-placed copy** — silently,
  while reporting success. This happened whenever two machines disagreed about a game's save root by
  even one path segment. A restore whose target is deeper than the archive's root is now refused
  before anything is copied or deleted, and the message names the repeated segment.
- **`<winPublic>` resolved one level too deep on Windows**, producing paths like
  `…\Public\Documents\Documents\…` for all 44 manifest games that use it. The Proton side was already
  correct — which meant a Windows PC and a Steam Deck disagreed about the same game's save root
  automatically, with no user error involved at all.
- **The Windows tray agent deadlocked on Exit.** Right-click → Exit did not stop the agent, and a
  second Exit left the menu frozen on screen; only Task Manager could end it. Present in every
  Windows agent since v0.1.8.
- **An update check that failed said nothing at all** — no toast, no log line. Every outcome is now
  logged, and a check you started always answers.

## Known issues

- **Steam Deck: the D-pad does not work** in the agent's folder browser. SteamOS Desktop Mode maps
  the right stick to the cursor and the left stick to scroll, and the D-pad to neither. Use
  point-and-click, or arrow keys with a keyboard attached.
- **Linux and Steam Deck have no auto-update.** Download the newer tarball and re-run `install.sh`.
- **Upgrading *to* v0.3.0 or later from v0.2.0 runs the old `install.sh`**, which could kill the
  running agent mid-upgrade while printing "Installed." Stop the agent first:
  `systemctl --user stop savelocker.service`. From v0.3.0 onward the script handles this itself.

## Note on v0.3.1

There is deliberately no v0.3.1 release. It was tagged, but its build assets were lost to a GitHub
incident on 19 July 2026 and it was never published — nobody ever ran it. By the time the incident
cleared, four more fixes had landed, two of them capable of destroying saves. Shipping a build with
known save-destroying behaviour when the fix already existed was not defensible, so v0.3.1 was
superseded by v0.3.2. The tag remains only as a history marker.
