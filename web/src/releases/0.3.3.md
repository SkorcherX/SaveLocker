# v0.3.3

_Released 23 July 2026_

One fix, for a bug that could make a single machine conflict with **itself** on every save — and, as
a side effect, quietly stop old saves from ever being cleaned up. If you have seen a conflict you
could not get rid of on a game only one machine plays, this is the cause.

**Every agent since v0.1.8 is affected, on both Windows and Steam Deck.** Upgrade.

## Fixed

- **A machine could conflict with itself on every save.** The agent is more than one process: a
  background service, plus a launcher that Steam starts alongside your game. The background service
  read its sync bookkeeping once when it started and never re-read it — so after the launcher
  uploaded your progress, the service was still describing an older save as its starting point. The
  server correctly refused that as a conflict. Worse, the service never caught up, so it conflicted
  again on **every** save until it was restarted.

  On a Steam Deck this looked exactly like two machines fighting over a game, except only one machine
  was playing it. Resolving the conflict in the console did not help, because the resolution never
  reached the agent.

- **Saves stopped being cleaned up on any game stuck in that state.** Version retention only runs
  after a save is accepted, and a conflicted save never is — while every version was also held back
  from deletion by the unresolved conflicts. One game reached **80 stored versions and 2.6 GB** while
  configured to keep 5.

- **Changing a setting could undo a save you had just uploaded.** Saving settings, editing a save
  folder, or adding a game wrote the background service's start-up view of your sync state back over
  the newer one. This is the same bug from the other direction, and it struck at the worst moment —
  while you were in the console fixing something.

## Upgrading

Windows agents update themselves. On Steam Deck, download the newer tarball and re-run `install.sh`.

If a game is currently stuck in a conflict loop, upgrade first, then resolve the conflict once in the
console and run `savelocker pull` on the affected machine. That is what re-points the agent at the
current save; without it the machine stays behind and conflicts again.

## Known issues

- **When two machines genuinely do conflict, the console shows only one conflict per game — the
  oldest.** If several pile up, resolving one reveals the next. Being improved.
- **A conflict alert can be dismissed without resolving anything.** Dismissing the notification does
  not resolve the conflict, and the conflict does not go away on its own.
- **Steam Deck: the D-pad does not work** in the agent's folder browser. SteamOS Desktop Mode maps
  the right stick to the cursor and the left stick to scroll, and the D-pad to neither. Use
  point-and-click, or arrow keys with a keyboard attached.
- **Linux and Steam Deck have no auto-update.** Download the newer tarball and re-run `install.sh`.
